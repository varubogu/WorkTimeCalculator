# 機能仕様

SES エンジニアの月次・年次作業時間を見積もり、目標範囲との差分を把握するツールです。

## 全体画面

- **デスクトップ**: 2 ペイン構成（左: サイドバー、右: ビュー切替）。
- **モバイル**: 縦積み構成。月ナビ・ビュータブ・メインビューの順。月の一覧と言語・テーマ・設定はヘッダ右側のハンバーガーメニュー内に表示。

ヘッダには「作業時間カルキ（ブランド）」「言語切替（JA/EN）」「読み込み」「書き出し」「ダークモード切替」「設定」があります。読み込み・書き出し・設定はアイコンボタンで表示します。モバイルでは言語切替・ダークモード切替・設定をハンバーガーメニューへ移動します。

## ビュー

タブで 2 種類のビューを切り替えます。

### カレンダー ビュー

- 当月のカレンダーを 7 列グリッドで表示。
- セルには日付と、記録がある日は実働時間が入ります。
  - 既定は時分表記（`7:30`、`8:00`）。
  - 設定で小数表記（`7.50h`）へ切り替えできます。
- セルの色・アイコンは種別（`kind`）に応じて変化：
  - **regular** (`reg`): 通常勤務（色なし）
  - **overtime** (`ot`): 残業。アイコンチップ表示
  - **vacation** (`vac`): 休暇
  - **weekend work** (`wknd`): 休日出勤
  - **holiday** (`holi`): 祝日（色付け）
  - **off** (`off`): 勤務なし
- 上部には月合計と、目標範囲（min–max）との差分チップ（[DeltaChip](../src/components/DeltaChip.tsx)）とプログレスバー（[RangeProgress](../src/components/RangeProgress.tsx)）。
- 月合計の下に、残業時間のみを対象にした「目標値」と「限界値」、差分チップ、プログレスバーを表示します。
- `定時を一括入力` ボタンで、表示中の月の平日へ定時勤務をまとめて登録できます。
  - 有効期間付き設定がある場合、その日の `dayHours` / `dayStart` / `breakMin` を使って日別に登録します。
- 土日・祝日・勤務なしの日もクリックして勤務記録を入力できます。

### 定時の一括入力

カレンダービューの `定時を一括入力` ボタンは、現在表示している月だけを対象にします。

- 対象日は月曜〜金曜のみ。
- 設定の「祝日を自動反映」が ON の場合、祝日は対象外。
- 保存される値は `{ start, end, brk, vac: false }`。
  - `start` は設定の `dayStart`。
  - `end` は `dayStart + dayHours + breakMin` から自動計算。
  - `brk` は設定の `breakMin`。
- 対象日のうち `start` / `end` / `vac` のいずれかが入っている日が 1 件でもある場合、上書き確認を表示。
- 確認でキャンセルした場合は何も保存しません。
- 終了時刻が翌日になる設定では、一括入力を中断します。

### チャート ビュー

- SVG 折れ線で 12 ヶ月分を可視化（[YearTimelineChart](../src/components/YearTimelineChart.tsx)）。
- 総作業時間チャートは目標範囲、残業チャートは「目標値以下=緑」「限界値以下=黄」「限界値超過=赤」で確認できます。
- 点にホバーするとツールチップ（合計・残業・休暇・目標）が表示。
- 点クリックでその月を選択（現在月は破線縦軸で強調）。
- 総作業時間の年次チャートに加え、残業時間のみを集計した年次チャートも表示します。
- 下部には年間の「定時 / 残業 / 休日出勤 / 休暇（日数）」サマリカード。

## 日単位の記録入力

カレンダーの任意の日セルをクリック → [DayModal](../src/components/DayModal.tsx) が開きます。土日・祝日・勤務なしの日も入力できます。

入力項目:

- **開始時間** (`HH:MM`)
- **終了時間** (`HH:MM`)
- **休憩（分）**（既定値は設定の `breakMin`）
- **休暇**（チェックすると時刻・休憩が無効化）

保存時、モーダル下部に自動計算された実働時間（`fmtH` 整形）が表示されます。表示形式は設定の「作業時間の表示形式」に従います。有効期間付き設定が該当する日は、その期間の `breakMin` / `timeStepMin` が初期値として使われます。

`クリア`ボタンで当該日の localStorage エントリを削除できます。

## 設定

ヘッダ右上の設定アイコンボタンで [SettingsModal](../src/components/SettingsModal.tsx) を開きます。

| セクション | 項目 |
|------------|------|
| Hours | 1日の定時時間（h）、定時開始時刻、時刻入力の刻み（分）、デフォルト休憩（分）、作業時間の表示形式 |
| Monthly target | 月目標の最小（h）、最大（h）、月残業目標値（h）、月残業限界値（h） |
| Yearly target | 年目標の最小（h）、最大（h）、年残業目標値（h）、年残業限界値（h） |
| — | 祝日を自動反映（トグル） |
| Effective periods | 初期状態では空欄の `開始日 ～ 終了日` を 1 件自動作成。開始日を設定すると元の期間を引き継いだまま新しい期間を分割して作成。プルダウンで対象期間を切り替えて編集し、`＋` から任意の期間も追加可能 |

保存すると `localStorage` の `wtc_settings` と `wtc_settings_periods` に書き込まれ、以後のセッションでも維持されます。

いずれの設定期間にも属さない日付が表示年に存在する場合、カレンダーとチャートの手前に「設定が存在しない期間があります」と警告が表示されます。

`1日の定時時間` は `7.5` のような小数入力に対応しています。時刻入力の刻みは 1 分以上で指定でき、15 分・30 分などの運用に合わせられます。

`作業時間の表示形式` は次の 2 種類です。

- **時分表記**: `7:30`、`8:00` のように `H:MM` で表示
- **小数表記**: `7.50h`、`8.00h` のように小数 2 桁で表示

## 目標範囲と判定

- **月目標**: `monthTargetMin` 〜 `monthTargetMax`（デフォルト 140–180h）
- **月残業目標**: `monthOvertimeTargetMin` は目標値、`monthOvertimeTargetMax` は限界値（デフォルト 0h / 45h）
- **年目標**: `yearTargetMin` 〜 `yearTargetMax`（デフォルト 1680–2160h）
- **年残業目標**: `yearOvertimeTargetMin` は目標値、`yearOvertimeTargetMax` は限界値（デフォルト 0h / 360h）

判定は 2 種類あります。

### 総作業時間の判定

| 状態 | 条件 | 色 |
|------|------|----|
| **ok / inRange** | `min ≤ value ≤ max` | 緑 |
| **over / excess** | `value > max` | 赤 |
| **under / shortage** | `value < min` | アンバー |

### 残業時間の判定

| 状態 | 条件 | 色 |
|------|------|----|
| **withinTarget** | `value ≤ target` | 緑 |
| **withinLimit** | `target < value ≤ limit` | アンバー |
| **limitExceeded** | `value > limit` | 赤 |

総作業時間と残業時間では判定ルールが異なり、[DeltaChip](../src/components/DeltaChip.tsx)、[RangeProgress](../src/components/RangeProgress.tsx)、[YearTimelineChart](../src/components/YearTimelineChart.tsx) でそれぞれに応じた色分けを行います。

## 祝日の扱い

- 設定の「祝日を自動反映」が ON のとき、表示年の日本祝日を Holidays JP API から取得します。
- 取得した祝日は `wtc_holidays_YYYY` として `localStorage` にキャッシュします。キャッシュは 7 日で再取得します。
- API 取得に失敗した場合は [holidays.ts](../src/holidays.ts) の同梱データをフォールバックとして使います。
- 祝日はカレンダーで色付けされ、勤務記録がない場合は種別（`holi`）を適用します。
- 祝日に勤務記録を入れると `ot`（残業）扱いになります。
- 残業時間の集計では、平日の `dayHours` 超過分に加えて、土日・祝日に勤務した時間はすべて残業として扱います。

## 言語・テーマ

- **言語**: JA / EN（[i18n.ts](../src/i18n.ts)）。選択は `Settings.lang` として永続化。
- **ダークモード**: `<body>` に `dark` クラスを付与し、CSS カスタムプロパティで配色を切替（[index.css](../src/index.css)）。

## ファイル入出力

- **読み込みボタン**: ヘッダのアップロードアイコンから「作業時間 CSV / JSON / YAML」「設定 JSON / YAML」を選びます。選択後のファイルダイアログは、その形式に合う拡張子へ絞り込みます。
- **書き出しボタン**: ヘッダのダウンロードアイコンから「作業時間 CSV / JSON / YAML」「設定 JSON / YAML」を選びます。
- **作業時間ファイルの読み込み**: 読み込み後、該当日の `localStorage` エントリを保存します。
- **設定ファイルの読み込み**: `baseSettings` と `periods` を検証してから適用します。
- **検証**: JSON / YAML はスキーマ構造に基づいて検証し、不正な内容はアラート表示して取り込みません。

## ナビゲーション

- **前月 / 次月**: ボタン or モバイルの `‹ / ›` で `monthIdx` を変更。
- **前年 / 次年**: `← {year-1}` / `{year+1} →` ボタン。
- **月選択**: サイドバー行、モバイルの月グリッド、または年チャートの点クリック。

## プライバシー

- 勤務データと設定はブラウザ外へ送信しません。
- 祝日自動反映が ON の場合のみ、祝日の日付リストを取得するために Holidays JP API へアクセスします。
- アナリティクスはありません。
- 端末をまたぐ移行は、作業時間ファイル・設定ファイルのエクスポート / インポートで対応できます。

## 未実装 / 既知の制限

- **日またぎ勤務（深夜越え）未対応**: `end <= start` の記録は 0 分扱い。
- **祝日は日本のみ**: 他国の祝日は未対応。
- **週単位の集計なし**: 現状は月・年の 2 階層のみ。
